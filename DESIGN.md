# スターカート (STAR KART) — ゲームデザイン文書

※ タイトルは「シュガリア・グランプリ」から「スターカート」へ改名(ユーザー指定)。
世界観・レース名としての「シュガリア王国」「シュガリアGP」はローカルの地名/大会名として存続する。

ブラウザで動く3Dカートレースゲーム。お菓子の王国「シュガリア」を舞台に、
お菓子から生まれた可愛いキャラクターたちがカートで競い合う。
※ ゲームシステムは定番カートレースを踏襲するが、名称・キャラ・デザイン・音楽は全てオリジナル。

---

## 1. タイトル・世界観

**タイトル: シュガリア・グランプリ (Sugaria Grand Prix)**

はるか雲の上、一年中お菓子作りの魔法が香る「シュガリア王国」。大地はスポンジケーキ、川は溶けたチョコレート、山脈はメレンゲでできており、住民たちはお菓子から生まれた不思議な生き物たち。年に一度の「シュガリア・グランプリ」は、国中の生き物たちが手作りカートで王国を駆け巡る一大祭典。優勝者は次の一年、王国の味を決める「最高パティシエ」の称号を得る。

- トーン: 甘くて可愛い、明るくにぎやか。パステル+ビビッドの菓子色
- コースの地形そのものがお菓子でできている(チョコの川、クッキーの橋、飴細工の城)

### モデルレシピ(Three.jsプリミティブ+Canvasテクスチャで構築)

全キャラ共通: 目は大きめの黒球(+白ハイライト)、口はCanvasテクスチャの弧線で「にこっ」と描き、表情を最優先。
体は丸みのあるプリミティブ構成、頭身は低く(1.5頭身)、手足は短いカプセル。

1. **マカロン・ププル**: 体=横に潰れた球2枚重ね(上下の殻)+間に白いクリーム円柱、頬=ピンク半球、頭上に小さいリボン(トーラス)
2. **ドーナ・リング**: 体=縦置きトーラス(いちごチョコがけテクスチャ+色付きスプレー粒)、顔=穴の中の球
3. **タプリン**: 体=円錐台(プリン形)、頭頂=とろけるキャラメルの平たい球、足元に白い皿(薄円柱)
4. **ソフクリン**: 頭=渦巻き状に積んだ円錐3段(白×水色マーブル)、下半身=コーン(格子テクスチャの逆円錐)
5. **ソーダ・シュワリ**: 体=薄緑の透明感ある球(雫形)、頭にビー玉(青い小球)、周囲に泡(白い極小球を数個浮遊)
6. **ワッフル・グリッド**: 体=角丸の箱(格子焼き目テクスチャ)+頭は半球で丸みを出す、頭頂にはちみつの雫(半透明円錐)
7. **ショコラ・ノワール**: 体=割れ目溝テクスチャの直方体、腰に金の帯、**目を大きく+口を描き表情を強調**(審査指摘対応)
8. **バウム・ロール**: 体=年輪テクスチャの円柱、頭=球+粉砂糖の白い小球、**大きな優しいタレ目**(審査指摘対応)
9. **ボンボン・キャノン**: 体=大きな球+**丸い半球の突起を少なめに配置**(審査指摘対応: 尖らせず可愛く)、赤白ストライプテクスチャ

### カート

共通形状(丸っこいボディ=角丸ボックス+カプセル、タイヤ=トーラス4輪、後部にマフラー円柱×2)。
ボディ色はキャラのメインカラー。キャラはカートの座席に乗せる。

## 4. コース(3本)

### ① クッキータウン・ロード(初級)
- **テーマ**: 焼きたてクッキーとビスケットの街並み。BGM=ほのぼの弾む街角ポップス調
- **レイアウト**: 緩いS字の商店街→大きな右コーナー→クッキー橋の緩い上り→広い直線→ゴール前の左複合コーナー。高低差小
- **実装ギミック**: ブーストパッド、小ジャンプ台、粉砂糖エリア(軽い視界演出)
- **ショートカット**: 橋の手前の狭い路地(縁石接触リスクあり、距離短縮小)

### ② チョコレート・キャニオン(中級)
- **テーマ**: 溶けたチョコの川が流れる渓谷。BGM=パーカッションが響く冒険オーケストラ調
- **レイアウト**: 連続ヘアピンで谷底へ下り→チョコの滝沿いの長い上り→ジャンプ台付きの下り九十九折れ。高低差大
- **実装ギミック**: チョコ噴水(踏むと一時加速)、チョコの川(落下=リスポーン)、揺れるビスケット筏(周期的に沈む足場)
- **ショートカット**: ビスケット筏ルート(大幅短縮、タイミングを外すとチョコ川へ落下)

### ③ レインボーシュガー・スカイキャッスル(上級)
- **テーマ**: 雲上の金平糖と飴細工の天空城。BGM=きらびやかで浮遊感のあるシンセ+鈴のファンファーレ調
- **レイアウト**: 細かい連続コーナー→巨大螺旋スロープで城上層へ→途切れた浮遊アメ玉足場のジャンプ地帯→急降下ロングストレート。高低差最大
- **実装ギミック**: ソーダの泡ゾーン(ふわっと浮く)、回転する飴細工の風車(接触で弾かれる)、コース外=雲海落下でリスポーン
- **ショートカット**: 外壁直登ルート(大幅短縮、失敗すると雲海へ落下)

全コース共通: アイテムボックス列(回転する星型キャンディ箱)、スタート/ゴールゲート、周回チェックポイント、ミニマップ用外形。

## 5. ゲームシステム

### 操作(キーボード)
| 操作 | キー |
|------|-----|
| アクセル | ↑ / W |
| ブレーキ/バック | ↓ / S |
| ステア | ←→ / A D |
| ドリフト(ホップ) | Space または Shift |
| アイテム使用 | E または Enter |
| ポーズ | Esc / P |

### ドリフト&ミニターボ(初期チューニング値、config.jsで管理)
- ドリフトキー+ステアでホップ→ドリフト突入。ドリフト中は曲率アップ+横滑り
- ステア量に応じてチャージ蓄積: 青火花(0.8秒)→橙(1.8秒)→紫(3.0秒)
- リリースでブースト: 0.5秒 / 0.9秒 / 1.4秒(速度×1.3)
- スタートダッシュ: カウントダウン「2」でアクセル押しっぱなし→成功でブースト

### レース
- 3周制、8台(プレイヤー+CPU7、ロスター9体から選択)
- 順位はスプライン弧長ベースでリアルタイム算出。最終ラップでBGMテンポアップ
- コース外(芝/砂糖)=減速。落下=数秒後にコース上へリスポーン
- カート同士は球衝突、重量差で押し負ける

### CPU AI
- レコードライン(スプライン+オフセット)追従、速度に応じた先読み点ステアリング
- ラバーバンド: プレイヤーとの距離に応じ速度±(後方時最大+12%、前方時-8%)で接戦を演出
- アイテムは種類に応じた簡易ロジックで使用(攻撃系は射程内に敵がいる時など)

### モード
- **グランプリ**: 3コース連戦、ポイント制(15/12/10/8/6/4/2/1)、総合順位で表彰式
- **タイムアタック**: 1人で走行、ベストタイムをlocalStorageに保存
- **シングルレース**: コースを選んで1レース

### HUD
順位(大きく)、周回数、所持アイテム(ルーレット演出付き)、ミニマップ(Canvas 2D、全カートの点)、ラップタイム、最終ラップ表示

## 6. 技術設計

### スタック
- Three.js **UMDビルドをローカル同梱**(`lib/three.min.js`、r134系をvendor) — index.htmlダブルクリックで起動可能
- Vanilla JS、ビルドツールなし。`<script>`タグの順次読み込み、`window.Game`名前空間で連携
- 全アセットはプロシージャル生成(プリミティブ+CanvasTexture)。外部画像・音源・fetchなし(file://で完結)
- 音はWeb Audio APIで合成(初回ユーザー操作で初期化)

### ファイル構成と読み込み順
```

lib/three.min.js      … Three.js UMD (vendored)

src/core/config.js    … 全チューニング定数(物理/アイテム/AI/抽選テーブル)

src/ui/hud.js         … HUD描画+ミニマップ
src/ui/screens.js     … 画面フロー(タイトル/キャラ選択/コース選択/リザルト/表彰式)

src/main.js           … 起動
```

### 主要インターフェース(サブエージェント委任時の契約)
- `Game.config` … 全定数。マジックナンバー禁止、必ずここに置く
- `Course`: `{ spline, halfWidth(t), surfaceAt(pos), checkpoints, itemBoxes, boostPads, hazards, minimapShape, respawnPoint(progress) }`
- `Kart`: `{ pos, heading, speed, update(dt, inputLike, course), applyBoost(), applySpin(), applyHit() }` — 入力は人間/AI共通の `inputLike` 抽象
- `RaceManager`: 周回・順位・進行度を管理し、`Game.state` に公開。HUD/AIはここだけを参照
- `Items`: `use(kart)`, `spawnProjectile()`, エンティティはRaceManagerのupdateループに登録
- 画面遷移: `Game.setState('title'|'charSelect'|'courseSelect'|'race'|'result'|'award')`

### 品質基準
- 60fps目標(最低30fps)。ジオメトリはマージ、パーティクルはプール化
- コンソールエラーゼロ。各フェーズ末にブラウザ実起動+スクリーンショット確認
- 「気持ちよく走れるか」最優先。仕様と手触りが衝突したら手触りを取る

## 7. サウンド設計(Web Audio合成)

- **BGM**: チップチューン風シーケンサ(矩形波+三角波+簡易ドラム)。タイトル曲、コース3曲(街ポップス/冒険調/きらきらファンファーレ調)、リザルトジングル。最終ラップはテンポ+半音上げ
- **SFX**: エンジン音(速度でピッチ変化)、ドリフトスキール、ホップ、ミニターボ解放、アイテムルーレット/取得、発射、被弾スピン、シールド防御、ブースト、カウントダウン(ピッ×3+ポーン)、ゴールファンファーレ、コース外走行音
